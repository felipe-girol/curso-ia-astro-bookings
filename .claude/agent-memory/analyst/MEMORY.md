## Conventions
- Issue titles: `type(module): short description` (e.g., `feat(launches): define types and DTOs`)
- Label: `enhancement` for features, `bug` for bugs
- Spec status flow: Draft -> Planned -> Coded -> Verified -> Released
- Double-link: spec file lists issues table, issues link back to spec file on GitHub
